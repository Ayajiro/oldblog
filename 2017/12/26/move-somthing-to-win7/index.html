2017-12-26

工作相關

其實我已經崩潰了

現在還會有人需要轉移到win7嗎？（困惑）

公司因為一些內部規定，新人電腦硬體全新但是規格低，搭配 win7 32bit，因此一直都是用額外的mbp在工作，不過有時候要用內部系統就得換來換去，前陣子拿到了規格稍微好一點的舊主機，跑起來相對順，因此決定把一些工作轉移到桌機上。

這次要轉移的是 sublime text 3 和 jupyter notebook ，但你只會在這篇看到 sublime text 3 簡短轉移的紀錄(因為有人寫啦)，和我的自言自語。

jupyter 那段完全卡死，只能記錄一下後來摸索出來的使用方法（對，裝完之後竟然一度開不起來）。

win7上的 jupyter notebook 使用撇步

原本想要用 cmd 安裝 jupyter 但可能是 path 設定問題所以失敗了，於是用了和 mbp 上一樣的方式，安裝 anaconda！

裝好之後發現anaconda啟動需要一點時間（因為硬體和系統的關係，大概需要五分鐘orz），進入之後開啟 jupyter 又需要一段時間，根本無法快速啟動 jupyter notebook ，還好anaconda已經想好了。

以上步驟從開到notebook跳出來，大概需要的時間是十分鐘左右，但還好不是緊急業務要使用，偶爾日常需要爬資料的時候才會開起來，不過自己寫的py內部設定還需要大量調整阿阿，到底要什麼時候才可以在win7上寫py阿…
